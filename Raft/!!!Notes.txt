------------- Typical task for RAFT fault-tolerance cluster
 - Distributed lock
 - Distributed Transaction coordination
 - Distributed counter
 - Distributed unique ID generator


 -----------Alg: Distributed counter

  - NodeClient with embedded Raft.NET sends command "Increase on 1 counter xyz" or "Increase on 1 counter xyz and get this value back"  
  - node redirects command to the leader and receives back its ticket number (term  index)
  - leader applies command 
  - when command is commited nodes start to receive event with last commited index
  - NC must be able can get all commands starting from index X (not including) up to (including) COMMITED index.
    All nodes apply command on local nodes:
	(increasing xyz value to 1)
	?Return
